Best practice: SignIn

The GameService module doesn't sign in automatically. You have to do it by yourself.

Providing a sign-in button

If the user starts the game for the first time, you should mention the advantages of your Game Services. For example show a message or display a signin button in the main menu.
When users click the sign-in button, your game should initiate the sign-in flow with the following command:

gameservice.beginUserSignIn()

If the sign-in process was successful, the command:

gameservice.isLoggedIn()

should return true. Hide the sign-in button and display a sign-out button. So the user have the ability to sign-out.

Signing the player in at startup

After users sign in successfully for the first time in your game, your game should sign them in automatically whenever they start the game again, until they explicitly sign out.

You have to save the sign-in state by yourself. For example with "SaveState" and "LoadState".

Checking if a user is signed in

You can call the

gameservice.isLoggedIn()

method to determine whether the user is signed in or not. 

Signing the player out

To initiate the sign-out flow, simply call 

gameservice.signOut()

Also, your game should save a state to indicate that the user explicitly signed out, then check that state the next time the game starts. Based on the saved state value, your game can then determine whether or not to call beginUserSignIn() to sign the user automatically in.
